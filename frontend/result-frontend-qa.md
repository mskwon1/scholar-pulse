## 2. Security & Bug Review (Step 7)

### Identified & Fixed Issues:
1. **Brittle DOM Access**: The `addKeyword` function in the dashboard used `previousSibling` to access the input value, which is prone to breaking with DOM changes.
   - **Fix**: Switched to using unique IDs (`kw-input-${tIdx}`) for keyword inputs to ensure stable access.
2. **Missing Input Validation**: `parseInt` was used on raw input values without `NaN` checks, potentially leading to invalid data storage.
   - **Fix**: Updated `updateFilter` to handle `NaN` by defaulting to `0` and improved state update patterns using spread operators.
3. **Immutability Patterns**: Keyword array updates were using `push` on a reference.
   - **Fix**: Refactored to use immutable array spreading (`[...old, newItem]`) for cleaner React state updates.

### Security Assessment:
- **Client-side Auth Guard**: Basic auth redirection logic is present in the dashboard.
- **Supabase Config**: Environment variables are used for project URL and Anon Key. No private service keys were found in the client-side code.
